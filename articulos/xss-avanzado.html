XSS Avanzado: Bypassing Filters

15 Enero 2023 10 min de lectura Bug Bounty

Introducción

Los filtros XSS son cada vez más sofisticados, pero con las técnicas adecuadas todavía es posible bypassearlos. En este artículo exploraremos métodos avanzados para evadir los filtros de seguridad más comunes.

Tipos de filtros XSS

Existen varios tipos de filtros que las aplicaciones web implementan para prevenir ataques XSS:

  • Filtros basados en blacklist
  • Filtros basados en whitelist
  • WAFs (Web Application Firewalls)
  • Sanitización de entrada/salida
  • Content Security Policy (CSP)

Técnicas de bypass para filtros basados en blacklist

Los filtros basados en blacklist suelen buscar patrones específicos como "script", "javascript:", "onerror", etc. Para evadirlos:

1. Uso de mayúsculas y minúsculas

<ScRiPt>alert('XSS')</sCrIpT>

2. Encoding de caracteres

<script>alert('XSS')</script>

3. Uso de caracteres nulos

<script%00>alert('XSS')</script>

Bypass de atributos HTML

Cuando los eventos tradicionales están bloqueados, podemos usar eventos menos comunes:

<img src=x onerror=alert('XSS')>

O usando el evento onload en iframes:

<iframe src="javascript:alert('XSS')" onload="alert('XSS')"></iframe>

Bypass de Content Security Policy (CSP)

Las CSPs mal configuradas pueden ser bypasseadas mediante:

  • Uso de redirecciones 302
  • JSONP endpoints
  • AngularJS CSP bypass
  • Google Analytics DOM XSS

Conclusión

Los filtros XSS continúan evolucionando, pero también lo hacen las técnicas para evadirlos. Es crucial realizar pruebas exhaustivas y mantenernos actualizados con las últimas técnicas de bypass.

La mejor defensa contra XSS es una combinación de validación de entrada, sanitización de salida, CSP adecuadamente configurado y auditorías de seguridad regulares.
